## 🎉 博客上线

历经环境配置与主题调优，个人站点**风声鹤唳的Island**今日上线。本博客以技术笔记与日常分享为主要内容，慢慢刻画自己的成长轨迹。

### 💡 搭建博客的初衷

关于要不要写博客，为什么要写博客，这段简单思考的背后却也暗藏着我的踌躇。我担心以自己现在的能力不足以胜任博客的搭建与维护工作，忧虑自己也许写不出有足够价值的文章来记录或展示成果。但再三思量后，我之所以能够做出肯定的决定，也是因为设想到了这其中隐性的长期收益：通过写博文实现"输出式学习"，将碎片化的知识点梳理成完整的文字，是复盘所学、查漏补缺的自我精进；而公开输出、坦诚分享，更是一场温柔的自我督促。我可以从中锻炼自己的语言组织和文字表达能力，通过记录来方便未来回顾，通过分享来帮助他人解惑，让每一次学习都落地生根，为个人发展开阔全新的视野。

### 📝 搭建过程

**1. 环境准备**

Hexo 依赖 Node.js，使用 nvm 管理 Node 版本可以避免后续的版本冲突问题。通过 hexo init 初始化项目后，整个目录架构井然有序：scaffolds 收纳文章模板、source 承载博文与静态资源、themes 专属主题管理，分工明确，职责清晰。

**2. 主题选型与定制**

我曾逐一对比 Next、Icarus 等多款热门 Hexo 主题，最终选择了 Butterfly。搭建全程，花费最多心力的便是主题细节的微调。并非配置复杂，而是乐于打磨每一处细节。优化导航栏双语排版、采用左文右图的卡片版式、定制淡蓝紫色作为页面背景配色、适配系统的昼夜明暗模式、调整移动端版式、打磨圆角界面与灵动加载动画……每一项小改动都实际优化了使用体验。

**3. 部署流水线**

依托 GitHub Actions 搭建轻量化自动化部署流程，可自动完成代码检出、依赖安装、静态资源构建、Pages 部署全流程。每次执行 git push 推送更新后，不到一分钟即可完成全站更新，部署流畅高效。

**4. 功能插件**

为让站点兼具实用性与氛围感，兼顾加载速度与浏览体验，我适配整合了多款轻量化插件，补齐站点功能，改善用户体验：

- **Giscus 评论**：舍弃 Disqus，利用 GitHub Discussions API 实现轻量化交互，页面加载轻盈，交流氛围纯粹干净。
- **本地搜索**：通过 hexo-generator-search 生成本地检索索引，前端直接解析，不依赖任何外部服务。
- **APlayer 音乐**：侧边栏嵌入网易云歌单，利用 pjax 的 detach/reattach 机制确保切页不中断播放。
- **Local Font Awesome**：将 Font Awesome 图标资源本地化加载，规避第三方 CDN 延迟，精简加载流程，让页面流转更加顺滑。

### 🎨 设计细节

博客的整体视觉围绕"轻快、干净、内敛"三个关键词展开。

**配色**：主色调选取晴空蓝（`#5b9bd5`），区别于 GitHub 黑和 Material Blue 的技术社区常见选择，传达一种不张扬的清爽感。橙金（`#e8a850`）作为点缀色，用于分页器、按钮 hover 态等小面积元素，在冷色基调中制造暖色跳跃。背景色使用了极淡的蓝紫底色调，比纯白多添一分柔和。

**首页布局**：选择 封面图居右、标题摘要居左。这个布局让每篇文章的视觉重心落在右侧的配图上，左侧文字信息区保持干净、不被图片干扰。若文章未设置封面图，卡片则不显示图片区域，仅保留纯文字，整体观感统一规整。

**暗色模式**：站点支持跟随系统自动切换明暗主题，同时保留手动切换入口，适配不同使用习惯。暗色模式下背景切换为深灰蓝，全局UI元素同步适配调整，弱化强光对比，保证夜间浏览的舒适度。

**排版与动效**：全站采用圆角UI设计，搭配全屏加载动画、页面平滑过渡效果，提升浏览流畅度。文章卡片在 hover 时有上浮 + 阴影加深的微动效，点到即止，营造灵活生动的交互氛围。

**Subtitle**：依托首页Banner打造视觉主氛围，下方副标题搭载打字机动态效果，循环轮播诗句，形成独特的风格特质。
